# Lobby Frontend

The React-based web interface for the Lobby platform, built with **Vite** and styled with Vanilla CSS.

## Features

- **Authoritative Gameplay**: Shares game logic with the server via WebAssembly (Wasm).
- **Matchmaking & Lobby**: Real-time updates via Socket.io.
- **Analysis View**: Post-game board review with scaling and theme options.
- **Responsive Layout**: Designed for seamless play across different screen sizes.

## Getting Started

### Prerequisites

- **Node.js**: (v18+ recommended)
- **Rust & wasm-pack**: Only required if building the game engine from source.

### Setup

1.  **Install Dependencies**:
    ```bash
    npm install
    ```

2.  **Build the Game Core (Wasm)**:
    Before running the frontend, ensure the Rust core is compiled to Wasm:
    ```bash
    npm run build:wasm
    ```

3.  **Start Development Server**:
    ```bash
    npm run dev
    ```
    The app will typically be available at `http://localhost:5173`.

### Key Commands

- `npm run dev`: Launch the Vite development server.
- `npm run build`: Generate optimized production assets in `dist/`.
- `npm run build:wasm`: Compile the shared Rust library from `/new_main/frontend/rust-wasm/` into WebAssembly.
- `npm run preview`: Locally preview the production build.

## Project Structure

- `src/components/`: Reusable UI components (Board, Lobby, Analysis).
- `src/wasm_pkg/`: The compiled Rust game engine (generated by `build:wasm`).
- `src/styles/`: Vanilla CSS design system.
