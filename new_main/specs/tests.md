# Testing Infrastructure

The project maintains a rigorous testing suite covering the game engine, backend services, and mobile utilities.

## Overview

The platform uses a "Multi-Tier" testing strategy to ensure reliability across three different languages and runtimes.

| Layer | Environment | Framework | Test Count (New) |
| :--- | :--- | :--- | :--- |
| **Game Engine** | Rust (Native) | `cargo test` | 52 |
| **Backend** | Node.js | `Jest` | 50 |
| **Mobile Helpers** | Java (Android) | `JUnit` | 57 |

## Rust Engine Tests (`rust-core`)
Testing focuses on the deterministic nature of the game rules.
- **Location**: `new_main/rust-core/src/engine.rs` (inline module).
- **Scope**: Movement validation, capture logic, edge cases (pinning paradoxes), and board setup constraints.
- **Execution**: `cargo test -p rust-core`

## Backend Tests (`backend`)
Testing focuses on authentication, data persistence, and distributed state.
- **Location**: `new_main/backend/tests/`
- **Suites**:
  - `auth.test.js`: JWT signing and guest ID uniqueness.
  - `sync.test.js`: Valkey pub/sub state replication.
  - `utils.test.js`: Logger verbosity and ID validation.
  - `gameplay.test.js`: BigInt sanitization and lobby logic.
- **Execution**: `npm test -- --forceExit`

## Android Tests (`frontend/android`)
Testing focuses on native utility logic for the Capacitor mobile app.
- **Location**: `new_main/frontend/android/app/src/test/java/com/dedal/app/`
- **Scope**: Coordinate geometry (Distance calculation, point-in-circle), color parsing, and rating adjustment math.
- **Execution**: `./gradlew test`

## Continuous Core Testing
Because the Rust core is shared between the backend (NAPI) and frontend (WASM), any change to `engine.rs` must satisfy the Rust test suite before it can be integrated into either environment.

> [!TIP]
> Use the coverage reports generated by `cargo test` to identify untested logic paths in new piece abilities.
